## テスト方針

- **事前情報禁止**: モデルには画像内容の事前情報を与えない
- **厳格判定**: 緩い合格基準は許容しない
- **設計図書準拠**: DESIGN_REVISION_v2.md の要件に基づく

### 合格基準

| 項目 | 基準 |
|------|------|
| primary_event | **必須一致** |
| 人数カウント | **必須一致** |
| 人物属性 (PAR) | 主要属性 **60%以上一致** |
| 車両検出 | 検出有無 **必須一致** |
| カメラ品質 | 問題あれば検出 |
| レスポンス時間 | **200ms以内** |

## 総合合格基準

| 画像 | 主要チェック項目 |
|------|-----------------|
| 1 | `none` イベント、誤検出なし |
| 2 | `human` 1人、PAR属性取得 |
| 3 | `human` 2人、複数人カウント |
| 4 | `human` 1人、手ぶら判定 |
| 5 | `none` イベント、誤検出なし |
| 6 | `vehicle` 複数台、人物誤検出なし |

### 全体合格条件

- 6画像中 **6画像合格** (primary_event一致)
- 人数カウント **5/6以上正解**
- PAR属性 **主要属性60%以上検出**
- レスポンス時間 **全て200ms以内**

---

## 重要な不合格条件

以下の場合は**即不合格**:

1. `none` 画像で人物/車両を誤検出
2. `human` 画像で人物を検出できない
3. `vehicle` 画像で車両を検出できない
4. 人数カウントが2以上ずれる
5. レスポンス時間が300msを超える
